Memory management is C++'s main distinguishing feature.

As I've told you, every programming language is a way of looking at a computer's memory.
When you *declare* a variable with an instruction like:

```
signed int a = 1;
```

what you're really doing is telling the compiler to take a 32-bit area of memory (an `int` is 4 bytes, remember?), associate it with the name `a`, and write the binary value into it:

 ```
 00000000 00000000 00000000 00000001
 ```

 As you can see, most of the variable goes unused, so if you're dealing with values below 65,535, it's better to use `short int`, which only takes up two bytes.
 In the same way, if you declare the string:

 ```
 const char* motto = "Cogito ergo sum";
 ```

you're asking the compiler to take a 16-byte area of memory, save its starting address in the variable `motto`, and then write into it the 15 characters of the phrase, plus one final character, with value `0`, marking the end of the string.
You can use this method if you know in advance how big the variables you'll be dealing with will be, but if instead you have to handle values of variable size (for example, user input), you have two options: either you reserve a generous amount of space in advance, or you allocate it on the spot, based on your actual needs.
The first case works fine if you have to handle a few small pieces of data, but if you have to handle many large variables, dynamic allocation, even though it's more complex to manage, is more efficient.

The `new` keyword lets you allocate a given amount of memory, and returns a pointer to the first address of the allocated area:

```
char* str = new char[length];
```

The `str` pointer now holds the address of the memory area allocated by the `new` instruction.
If we pass it as a parameter to the `<<` operator, since it's a pointer to `char`, it will be displayed as a string:

```
cout << str << endl;
```

To find out its actual value, we need to convert it into a `void` pointer with a <a href="/en/data-types#casting">casting</a> operation:

```
cout << (void*)str << endl;
```

The downside of dynamic memory allocation is that, while a static buffer "lives" only as long as the block of instructions in which it was created, dynamically allocated memory stays occupied until it's freed with a `delete` instruction:

```
delete[] str;
```

As you can see, the globally-defined string is visible both in the `main` function and in the `func` function; the variable defined at the start of the `func` function is visible within the function itself, but not in the code block, where the new `str` variable is visible instead.
Each variable's "life expectancy" depends on where it was defined: the first variable will exist for the entire duration of the program; the second variable is created when the `func` function is called, and is destroyed when the function ends; the variable inside the code block only exists for two instructions, then the code block ends and it's destroyed.
By contrast, when a memory area is allocated dynamically, it stays occupied until the program (or, more precisely: the programmer) releases it with a `delete` instruction.
If the program makes only a single call, as in our example, the fact that a few dozen bytes aren't available for a few minutes doesn't cause much of a problem (I know this for a fact, because, in the first version of the example, I forgot to add the `delete` instruction at the end, and the computer kept working just fine anyway); but if that same program kept running on the same computer for a long time, it would gradually use up all the available memory, causing it to freeze.
This is why the language with the "J" has a *garbage collection* system that, like Secret Service cleanup crews, takes care of erasing the evidence of its programmers' incompetence before it can do any damage to the systems.
Someone will tell you that's not true, that "J" programmers are competent professionals, but think about it: if there's a garbage collection system, there has to be someone producing the garbage, doesn't there?

Like C++ variables, human beings too are called upon to live, carry out their task, and are eventually removed from the system.
The resources we occupy are a set of *spazioni*, rather than sequences of bytes, but what we commonly call "ourselves" is nothing more than a way of looking at the transitions of energy within the system.
Unlike software variables, though, human beings are able to evaluate the outcome of their choices or actions, and can decide whether to repeat that choice or action in other cycles of the Universe's life.
We are, at the same time, part of the program, its *beta testers*, and its developers, and if we notice a faulty instruction, we can fix it in subsequent "runs" of the program, the same way a chess player avoids repeating a move that has proven to be a loser.
This way, we generate a new variant of history which, like a chess variation, may turn out better or worse than the previous ones.
A very famous *Koan* speaks of <i id="istante">Zen in every instant</i>:

Tenno, confused, did not know the answer, and realized that he was unable to carry Zen every minute.

Another difference between human beings and C++ code is our relationship with the `delete` operator.
Mankind has always been afraid of death, and, ever since Gilgamesh, has always tried to escape it or undo its work.
Art, religion, science: we could say that everything good we have produced as a species stems from a natural refusal to accept the transience of our existence.
Death has been the cause, and at times the subject, of the finest fruits of our ingenuity.
Over time, all these creations have built a cultural cocoon that, on the one hand, has protected us from fear, and on the other, has given us shared ideals to fight for, even if not always for good ends.
In the last two centuries, though, Science has torn through this cocoon, stripping human beings of their comforting lies and replacing them with unsettling certainties, while its dim-witted child (the '68 Movement) swept away parts of our Society that were certainly in need of revision, but fundamental all the same, like the family or the school, without giving anything back in return, except the miniskirt.

Humanity, faced with the inescapability of the end and, at the same time, stripped of the comfort of religion and the support of family, has reacted like a patient diagnosed with an incurable illness, and has processed its grief according to the five stages defined by Dr. Kübler-Ross:<a class="nota" href="/en/notes#fn-kubler" id="kubler">2</a>

<i>Denial:</i> just as the Victorian Age had its taboo around sex, our "culture" has its taboo around death: our great-grandparents pretended to have no carnal interests, we pretend death doesn't exist.
We deny it in words, using euphemisms to avoid naming it: "He's gone", "He's no longer with us", "He's passed away", as if we were talking about an escaped convict or a magician.
We deny it in practice, isolating the dying in hospital wards, far from their homes and their loved ones.
We deny it in our thoughts and our actions, because otherwise the senselessness of our lives, spent chasing the fleeting mirage of success,<a class="nota" href="/en/notes#fn-successo" id="successo">3</a> would become obvious and undeniable.

<i>Anger:</i> although anger and aggression have always been present throughout our history, since the middle of the last century they have begun to flourish not only from oppression and hardship, but from prosperity as well.
The clashes between *Mods* and *Rockers* in the '60s, the armed struggles of the '70s, *Punk*, and, more recently, *foreign fighters* and aggression on *social media*: none of these phenomena arise from ghettos or from a state of need; they are all hobbies of the Middle Class.

<i>Bargaining:</i> in 1982 Jane Fonda released a videotape<a class="nota" href="/en/notes#fn-aerobica" id="aerobica">4</a> teaching aerobics in your own living room to anyone who could afford to spend sixty Dollars<a class="nota" href="/en/notes#fn-dollari" id="dollari">5</a> on a VHS.
It became one of the best-selling videotapes of all time, and kicked off a trend that turned into a lifestyle for millions of people.
The fitness craze, together with the resurgence of *New Age* spirituality, was how civilized peoples tried to come to terms with the terrifying new reality Science had presented them with, acknowledging their own mistakes and trying to remedy them by changing their lifestyle.
Unfortunately, though, the pseudo-mystics never understood that reciting the *Lotus Sutra* because you believe it can help you fulfill your desires is a contradiction in terms.
Religion is like antibiotics: it doesn't work if you don't take the full course.

<i>Depression:</i> depressive disorder is the second most common illness in the World, after heart disease — clear evidence that the bargaining phase hasn't produced the hoped-for effects.
Having failed every attempt to fight reality, the Man of the third Millennium sought refuge in flight, and shut himself away like Tokugawa-era Japan, but without the flourishing of the arts that accompanied *Sakoku*,<a class="nota" href="/en/notes#fn-sakoku" id="sakoku">6</a> because Reality is not a predator we can hope to shake off.

Just as every instant matters to the World, every being matters to the Universe, so long as it carries out the task assigned to it — even someone or something that seems insignificant.
It's well known that Master Canaro reached his first state of Enlightenment when he saw a shrub growing on a rock face overlooking the sea.
A seemingly useless shrub: its seeds would never reach soil where they could sprout, and its branches were too thin for a bird to build a nest in them; and yet, the sheer stubbornness with which that plant carried out the role assigned to it let Master Canaro understand something many books had failed to explain to him, laying the first stone of what would later become his teaching.
What we perceive as our "worth" is relative.
Think of the variable `a` we saw at the beginning:

```
00000000 00000000 00000000 00000001
```

The first bit is worth `0`, a null value, if you consider it on its own, but if instead you evaluate it in relation to the two bytes it belongs to, it becomes the value <a href="/en/data-types#modificatori">the variable's sign depends on</a>.
If, by some chance, the value of that bit became `1`, the value of the variable `a` would become negative, with unpredictable consequences for the program.
Many, too many people are
<a href="https://canaro.net/ide-monogatari/binario.html" target="canaro">like that bit</a>, and think they're worth zero because they measure their own worth only in subjective terms.
Mesmerized by the Age-Old Dichotomy, they spend all their energy trying to be worth `1`, and in doing so lose sight of what would have been their destiny — with consequences far more serious than a sign flip in a variable.

A good programmer must always check that his code has no flaws, and what I've just told you has one, quite a serious one: it can be used as an excuse to create a caste system that limits the growth potential of those who are genuinely destined to increase their worth.
The question, then, is: how can a bit, or a sentient being, decide what its exact worth is?
Certainly no established Authority can tell it, and neither can custom.
Family or school can offer suggestions, but they can't decide for it: parents and teachers surely have more experience, but they aren't infallible.
The answer lies in a Roman dialect proverb:

> First you gotta know why you're in this world.
Once you know why, you gotta learn how to be in it.

These two lines hold the meaning of Life within them: understanding what your role is, and carrying it out to the best of your ability, without letting yourself be swayed by fashions, and above all without falling victim to the Age-Old Dichotomy.
C++ is a strongly-typed language,<a class="nota" href="/en/notes#fn-tipizzazione" id="tipizzazione">8</a> so, when you declare a variable, you always have to give it a data type:

```
bool    boolean  = false;
char    letter   = 'C';
int     integer  = 1234567890;
float   decimal  = 3.14;
char    array[]  = "abcdefghilmnopqrstuvz";
```
In the same way, everything that exists has abilities suited to its role in the Universe.
Playing music, writing, acting, teaching, persuading, leading, loving: to each of us, the Universe gives a tool and a workbench, even if the role we've been assigned isn't always obvious from birth.
Take Lech Wałęsa, for example (he's the only Nobel Peace laureate I can think of who didn't have a degree): he started out working as a ship's electrician, but his abilities turned him into a political leader.<a class="nota" href="/en/notes#fn-walesa" id="walesa">9</a>
Or, better yet, think of yourself: at eighteen, if you'd wanted to, you could have left the Order, gone home, and become some kind of *deputy-something* at your family's company, but instead you stayed here writing your book.
Was that the right choice?
What did the Universe expect of you? That, as a monk, you'd make the World a better place with your Bible for geeks, or that, as a manager, you'd improve the living conditions of your company's employees?
We'll only find out once we go back to being One; until then, we can only speculate.<!--, oppure chiederci: quale è stato, il motivo della tua scelta?-->
If you stayed here out of laziness, or because you were afraid of the World or of your family, you made a mistake.
If you stayed because you don't desire luxury or power, you may have made the right choice. Maybe.
If you stayed, even though it would have been easier and more comfortable to return to the bosom of your family, because you thought you'd be more useful here, then you're either terribly naive or you did the right thing.
Either way, you should always do your own job well, whatever it is, because we never know how or when we'll be called upon to be useful.
Once, a friend of Master Canaro's, just back from Japan, told him:

> The difference between us and the Japanese is that if you put someone here to clean toilets, he feels looked down on and does a poor job, while a Japanese person tries to become the best toilet cleaner in the whole Country.

Another way to tell whether you're on the right path is through love.
Love is the perfect drug: it's as addictive as *crack* and makes you believe you can fly, like LSD; it's the honey with which Life lures the ants of our thoughts, making them go wherever it wants.
So, if what drives you to act is some form of love &mdash; and not just love for someone, but also for something, whether it's the sea, the mountains, a sport, a form of art, or a job &mdash; it's very likely that you're doing what you're meant to do.
From the standpoint of memory management, love has another use too, summed up in this precept:

> We love what will kill us (if all goes well)
