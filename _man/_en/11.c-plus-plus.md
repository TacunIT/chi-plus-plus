---

C++ is a programming language created by Bjarne Stroustrup in 1983, while he was working at AT&T Bell Labs.
Ten years earlier, his colleague Dennis Ritchie had created a programming language called *C*.
C, as I've told you, was extremely effective if you had to program computers, but &mdash; like some of your fellow brothers &mdash; it didn't handle real-life entities nearly as well.
Stroustrup, who during his doctorate had worked with an object-oriented language called *Simula67*, thought that if he could add to the execution speed of C the ability to create new data types like Simula's, he would get the perfect language.
He was right.
The name *C++* refers to the `++` operator, used to increment the value of a variable by one:

```

```
If you compile and run this code, you'll get:

```

> ../out/esempio

Valore di C = 13
```

C++, in fact, wasn't a new language: it was an improved C.
All the code and experience that had been built up around C until then could also be applied to C++.
Paraphrasing Neruda, Stroustrup did to C what Jesus did to Judaism: just as the Nazarene took a stern religion, suited to a people fleeing through the desert, and improved it, making it less authoritarian, the Dane transformed a language conceived solely for managing computers into a language that could manage anything.

---

The main additions C++ brought to C are: *data abstraction*, *object-oriented programming*, and *generic programming*.
Let me explain what these are now, but don't worry if you don't fully understand: we'll go deeper into all these concepts later.
C's data types are:

    char, int, float, double, array, enum, struct, union

More than enough to write the Unix *kernel*, but decidedly inadequate for writing a program to manage phone lines.
For a programming language to handle a data stream, a user, a phone line, or even a horse-breeding farm with equal ease, it needs to be able to manage, in addition to its predefined data types, new entities defined by the programmer.
This is what's called *data abstraction*, and C++ achieves it by means of *classes*.
Classes are the in-code representation of an entity:

```
class Cavallo {
private:
    Sesso  _sesso;
    string _razza;
public:
    const char* getSpecie() const {
        return "cavallo";
    }
    const char getSesso() const {        
        return (char)_sesso;
    }
    const char* getRazza() const {
        return _razza.c_str();
    }
    Cavallo() {}
    Cavallo(const char* razza, Sesso sesso ) {
        _razza  = razza;
        _sesso  = sesso;
    }
};
```

or of a concept:

```
class Monta {
private:
    Cavallo _maschio;
    Cavallo _femmina;
    Data     _giorno;
public:
    Monta(Cavallo& maschio, Cavallo& femmina) {
        _maschio = maschio;
        _femmina = femmina;
        time(&_giorno);
    }
};
```

Thanks to classes, the programmer can create new data types and use them within their program the same way they would use the language's primitive data types.
Each class has *attributes* and *methods*.
*Attributes* are data that describe the characteristics of the class, for example, a horse's breed or sex.
*Methods* are functions that define how the class can interact with the other elements of the program.
In the classes in the example, the *attributes* are the elements you see in the `private` section, while the *methods* are the functions you see in the `public` section.
The function with the same name as the class is called the *constructor* of the class, because it "explains" to the compiler how objects of this class should be created.
Classes, however, are the recipe, not the dish.
To be used, classes must be *instantiated* into objects:

```
int main()
{
    Cavallo stallone("lipizzano", maschio);    
    Cavallo giumenta("maremmano", femmina);
    Monta   monta(stallone, giumenta);
    cout << monta << endl;
    return 0;               
}
```

`stallone`, `giumenta`, and `monta` are three objects.
The first two are instances of the `Cavallo` class, the third is an instance of the `Monta` class.
If you add a bit of code to the classes we saw earlier and compile the program, you'll get:

```

For a programming language to call itself *object-oriented*, though, besides classes it must also be able to handle *inheritance* and *polymorphism*.
*Inheritance* allows you to define new classes as an extension of existing classes:
<a id="cast"></a>

```
class Animale {
private:
    string _specie;
    string _razza;
    Sesso  _sesso;
public:
    const char* getSpecie() const {
        return _specie.c_str();
    }
    const char getSesso() const {        
        return (char)_sesso;
    }
    const char* getRazza() const {
        return _razza.c_str();
    }
    Animale() {}
    Animale(const char* specie, const char* razza, const Sesso sesso ) {
        _specie = specie;
        _razza  = razza;
        _sesso  = sesso;
    }
};

class Cavallo : public Animale {
public:
    Cavallo() {}
    Cavallo(const char* razza, const Sesso sesso )
    : Animale("Cavallo", razza, sesso ) {
    }
};
```

In the example above, we first defined an `Animale` class, which has three attributes: `specie`, `razza`, and `sesso`; then we defined a `Cavallo` class, deriving it from the `Animale` class.
This way, if besides horses our program also had to handle other hoofed animals, we wouldn't have to repeat the same instructions in each class, but could reuse those of the base class:

```
class Asino : public Animale {
public:
    Asino() {}
    Asino(const char* razza, const Sesso sesso )
    : Animale("Asino", razza, sesso ) {
    }
};
```

At this point, your sharp mind should have caught a possible problem (assuming you're paying attention, which I'm not entirely convinced of): the `Monta` class can only handle objects of type `Cavallo`.
A language without polymorphism would force us to write two new classes: one for mules and one for hinnies:

```
class MontaMulo {
private:
    Asino   _maschio;
    Cavallo _femmina;
    Data     _giorno;
public:
    MontaMulo(Asino& maschio, Cavallo& femmina) {
        _maschio = maschio;
        _femmina = femmina;
        time(&_giorno);
    }
};

class MontaBardotto {
private:
    Cavallo _maschio;
    Asino   _femmina;
    Data     _giorno;
public:
    Monta(const Cavallo& maschio, const Asino& femmina) {
        _maschio = maschio;
        _femmina = femmina;
        time(&_giorno);
    }
};
```

The problem is that the more code you write, the more likely you are to make mistakes, and the harder they'll be to fix.
By contrast, programs with fewer lines of code are more reliable and easier to fix or modify.
C++ helps here because it allows for <i id="polimorfismo">polymorphism</i>, that is, the ability of a function or operator to carry out its task independently of the data type it has to handle.
If we rewrite the `Monta` class using, instead of parameters of type `Cavallo`, parameters of the base class type `Animale`:

```
class Monta {
private:
    Animale* _maschio;
    Animale* _femmina;
    Data     _giorno;
public:
    Monta(Animale* maschio, Animale* femmina) {
        _maschio = maschio;
        _femmina = femmina;
        time(&_giorno);
    }
};

```

As I've told you, though, a good programmer isn't satisfied just because they've managed to produce the expected result — they always ask whether there's a more efficient way to get there.
In our case, the code we used to display the results of the matings:

```
Monta puledro(cavallo, giumenta);
Monta ciuco(asino, asina);
Monta mulo(asino, giumenta);
Monta bardotto(asina, cavallo);

cout << "PULEDRO\n"  << puledro  << endl;
cout << "CIUCO\n"    << ciuco    << endl;
cout << "MULO\n"     << mulo     << endl;
cout << "BARDOTTO\n" << bardotto << endl;
```

isn't the most efficient, both because we could get the parents' species mismatched with the offspring's name, and because the instructions have to be repeated for each object.
To fix the first flaw, we can add to the `Monta` class an attribute and a method to independently determine what kind of offspring results from the mating:
<a id="membro"></a>
```
string   _esito;
void setEsito() {
    if(strcmp(_maschio->getSpecie(),"Asino") == 0) {
        if(strcmp(_femmina->getSpecie(),"Asino") == 0){
            _esito = "asino";
        } else {
            _esito = "mulo";
        }
    } else {        
        if(strcmp(_femmina->getSpecie(),"Cavallo") == 0) {
            _esito = "puledro";
        } else {
            _esito = "bardotto";
        }
    }
}
```

but even so, we'd still have to rewrite four lines of code to change the program's output: an unacceptable approach for production systems, where the entities being handled can number in the thousands.
We can solve this problem thanks to *generic programming* and the way it's implemented in C++: *template* classes:

```
template < class T> class list;
```

The `list` class is one of C++'s *template* classes, and lets you insert, remove, move, merge, sort, and list a collection of objects of the same class.
The syntax for creating a list of `Monta`-class objects is:

```
list<Monta> monte;
```

Once that's done, we can add elements to our list with the `push_back()` function, passing it the class constructor directly:

```
monte.push_back(Monta(cavallo, giumenta));
monte.push_back(Monta(asino, asina));       
monte.push_back(Monta(asino, giumenta));     
monte.push_back(Monta(cavallo, asina));
```

This code is a more efficient form of:

```
Monta m1(cavallo, giumenta);
Monta m2(asino, asina);       
Monta m3(asino, giumenta);     
Monta m4(cavallo, asina);

monte.push_back(m1);  
monte.push_back(m2);  
monte.push_back(m3);
monte.push_back(m4);
```

To display the contents of the list, regardless of the number of elements, all it takes is the instruction:

```
for (list<Monta>::iterator it=monte.begin(); it!=monte.end(); it++) {
    cout << *it << endl;
}
```

Our program's `main()` function will therefore be:

```
int main()
{
    Animale* cavallo  = new Cavallo("lipizzano", maschio);    
    Animale* giumenta = new Cavallo("maremmano", femmina);    
    Animale* asino    = new Asino("amiatino", maschio);
    Animale* asina    = new Asino("sardo", femmina);
    list<Monta> monte;
    monte.push_back(Monta(cavallo, giumenta));
    monte.push_back(Monta (asino, asina));       
    monte.push_back(Monta (asino, giumenta));     
    monte.push_back(Monta (cavallo, asina));
    for (list<Monta>::iterator it=monte.begin();
         it!=monte.end(); it++) {
        cout << *it << endl;
    }
    return 0;               
}
```
