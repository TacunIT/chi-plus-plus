The variety of available data types is C++'s main distinguishing feature.

C++ inherited a wide range of data types from C.
Depending on their nature, we can divide these data types into two groups: *scalar* data and *aggregate* data.
The *scalar* data types are: `void`, `bool`, `char`, `wchar_t`, `int`, `float`, `double`, and they let you handle a program's classic elements: integers, decimal numbers, letters, and boolean values.
The *aggregate* data types are *arrays*, *unions*, *enums*, *structs*, and *classes*.
These, as their name suggests, are made up of the aggregation of several scalar data items of the same type or of different types.

The `void` data type has neither a size nor a value, and can only be assigned to a pointer or to a function that returns no value at all, like the `colorName` function of the `Color` class.
The `bool` data type is used to handle boolean values.
It can only take two values, `true` or `false`, so it has the smallest possible size, namely one byte.
The `int` type lets you handle integers.
It's usually four bytes in size, which can be increased or decreased by means of keywords called
<a href="#modificatori" class="xref">modifiers</a>.
The `float` and `double` types are used to handle decimal numbers.
They are four and eight bytes long respectively, with a precision of seven and fifteen decimal digits.
The three elements that appear outside the `main` function are *aggregate* data types, and let you define non-standard data types.
`enum`s let you assign names to values, making programming easier.
In the example, the `RGB` *enum* assigns a name to three values of the RGB encoding.
This new data type is used in the `ColorRGB` struct.
`struct`s let you create sets of heterogeneous data.
The `ColorRGB` variable contains two different variables: a value of type `RGB` and a
<a href="#puntatori" class="xref">pointer</a>
to a character.
All the data types we've seen so far are a legacy of the *C* language.
The `Color` class, on the other hand, is one of the innovations introduced by C++, and contains two elements: a struct of type `ColorRGB` and a function that displays the color's name.
Be careful, though: none of these fellows is really what it claims to be.
The `boolean` variable doesn't actually contain the value `true` or `false`, but a binary value that the system treats as such.
Likewise, the `character` variable doesn't contain the letter `C`, but the binary value corresponding to the ASCII encoding of the letter `C`.
Every data type is nothing more than the name of a quantity of memory.
Remember this, because it's the key to everything.

The number of possible values a data type can hold depends on its size.
A `char` is made up of eight bits, and can therefore take 2<sup>8</sup> values, that is: from 0 to 255.
An `int` is made up of thirty-two bits, and can take 2<sup>32</sup> values, that is: from 0 to 4,294,967,295.
A `float` has 8 bits for the exponent and 23 for the mantissa, so it can take values from 1.175494e-38 to 3.402823e+38, and so on.
<a id="modificatori"></a>
Besides data types, C++ has also inherited from C the so-called *modifiers*, keywords which, when added to the name of a data type, change its characteristics.
The `short` and `long` modifiers change the size of the data, shrinking or growing it.
The `signed` and `unsigned` modifiers determine whether the first bit of the data is treated as its sign: positive if the value is 0, negative if the value is 1.
The sign, however, "steals" a bit from the *signed* variable, which can therefore hold fewer values than the same *unsigned* variable.

<a id="funzioni"></a>
The last thing I need to tell you, about C++'s data types, are the so-called *intermediary attributes*: *functions*, *arrays*, *pointers*, and *references*.
We'll talk about *functions* at length
<a href="/en/functions" class="xref">later on</a>.
What matters to us here and now is that every function has its own data type.
The `main` function, which is the main function of every program, has data type `int`, and indeed it ends with the instruction:

```
return 0;
```

The `nome_colore` function, inside the `Colore` class, is of type `void` and, as you can see, has no return value.
*Arrays* are sets of homogeneous data, and are declared by adding square brackets to the variable's name.
An array's size must be defined at the moment it's declared; this can happen either explicitly, by putting the number of elements inside the square brackets:

```
char elementi[10];
```

or by assigning the array a value that will determine its maximum size, as we saw in the first example:

```
char array[] = "abcdefghilmnopqrstuvz";
```

Each element of the array is identified by an index ranging from 0 to the number of elements in the array minus one.
In the case of the array above, the letter `a` will have index 0, `b` will have index 2, and so on, up to `z`, which will have index 20.

As you can see, the bits are the same, but by changing the data type associated with the pointer, and therefore its size, the value of the data becomes something else, and the conversion of numbers into letters loses its meaning.
The `p1++` and `p2++` instructions increment the pointer by one, that is, they move it forward in memory by a number of bytes equal to the size of the data type it's associated with.
The `p1` pointer is an `unsigned char`, so it moves by eight bits; the `p2` pointer is a `short int`, so it moves by sixteen bits.
This is why the second reading loop takes half as long as the first: because the amount of bits read per iteration is doubled.
If I'd been careless and copied the first reading loop's code as-is, without dividing the number of iterations by two, the pointer would have kept "walking" forward in memory, past the boundaries of the array, and this, as we'll see later, is definitely not good.
*References* are aliases for the variables they're associated with.
As with gibbons or swans, the bond between a *reference* and its variable lasts a lifetime.
While pointers can have a `NULL` value and can be associated with different variables over the course of processing, *references* must always be associated with a variable, and that association can never be changed afterward.

<hr id="casting">

Since each data type is nothing more than a way of looking at memory, it's possible to convert from one type to another.
Conversions can be *implicit* or *explicit*.
*Implicit* conversions are the ones that happen when a value is copied between variables of compatible type:

```
int  i = 0;
char c = i;
```

In this case, we assign to a variable of type `char` the value of a variable of type `int`, and the compiler lets us do it, because the value of the integer can be stored without any trouble in the single byte of the `char`.
If we wanted to make this conversion explicit, we would have to use the syntax:

```
int  i = 0;
char c = (char) i;
char c = char (i);
```
The two forms are equivalent.
The first is the one C++ inherited from C; the second is called *functional notation*.
In the earlier example, the `valori` variable is a pointer to `unsigned char`, but we told the system to treat it as a pointer to `short int` with the instruction:

```
duebyte * p2 = (duebyte*)valori;
```

C++ has other ways of converting one data type into another too, but since they apply mainly to classes, we'll see them later on.
