Today I'll tell you about *streams*, which, as is well known, are C++'s most important component.

C++ inherits from C the lack of dedicated keywords for I/O management.
Instead of instructions like BASIC's `print`, it uses libraries of classes and functions that let you convert the objects a program manages into printable text, or convert text elements into objects.
It couldn't be otherwise: C++ doesn't just have to handle strings and numbers, like BASIC, but also floating-point numbers, pointers, and above all user-defined data types, for which it wouldn't be possible to define a standard behavior, and which would therefore need to be handled differently from primitive data — so much for the consistency of the language.
Besides being able to make use of C's function libraries, C++ has its own I/O library, based on the `stream` class hierarchy, which lets you handle user-defined data types too.
We've already seen examples of this feature when we talked about <a href="/en/c-plus-plus#polymorphism" class="xref">polymorphism</a> and <a href="/en/polymorphism#overload-operatori" class="xref">operator overloading</a>:


```
ostream& operator << (ostream& os, const Animale& animale) {
    os  << "Specie:" << animale.getSpecie() << "\t"
        << "Razza:"  << animale.getRazza()  << "\t"
        << "Sesso:"  << animale.getSesso()  
        << endl;
    return os;   
}
```

This code "teaches" the `<<` operator how to behave when displaying an object of class `Animale`.
The same can be done (and we did it) for any other user-defined type.
It's the language's syntax that adapts to the programmer's needs, not the other way around.
A few key concepts for understanding streams are:

- a *stream* is an abstraction representing the source or destination of a set of data of variable length: keyboard input, screen output, memory buffers, strings, files;
- output to a stream, toward any destination, is called *writing* or *insertion*, and is performed by means of the `<<` operator;
- the terms *reading* or *extraction*, instead, refer to the operation of acquiring data from a source, performed by the `>>` operator.

C++'s `iostream` library lets you handle stream I/O operations by means of classes derived from two base classes: `streambuf` and `iosbase`.
The library has two different "family lines": one meant for handling single-byte characters, and one meant for multi-byte characters.
The classes of the multi-byte library have the same name as the ordinary classes, with the prefix "w" added.
This is the inheritance layout of the `iostream` library's classes:

```
                              ios_base
                                 |
                             basic_ios
                                 |
               __________________|_________________       
              |                                    |
              |                                    |
        basic_istream                         basic_ostream
             | |                                  | |   
             | |                                  | |  
             | |__________________________________| |  
             |                   |                  |   
      _______|______             |         _________|_______       
     |              |            |        |                 |
basic_istringstream |            |   basic_ostringstream    |
                    |            |                          |
                    |            |                          |
               basic_ifstream    |                  basic_ofstream
                                 |
                            basic_iostream
                         ________|________
                        |                 |
               basic_stringstream    basic_fstream

```

Except for `ios_base`, these are all template classes, later instantiated with different parameters to handle the character types `char` and `wchar_t`.
For example, the `ostream` class is a specialization of the `basic_ostream` class:

```
typedef basic_ostream<char> ostream;
```

Its multi-byte counterpart is the `wostream` class:

```
typedef basic_ostream<wchar_t> wostream;
```

The `basic_ostream` template class, in turn, derives from `basic_ios`:

```
template<
    class CharT,
    class Traits = std::char_traits<CharT>
> class basic_ostream
: virtual public std::basic_ios<CharT, Traits>
```
which, in turn, derives from `ios_base`:

```
template<
    class CharT,
    class Traits = std::char_traits<CharT>
> class basic_ios
: public std::ios_base
```

In short: if the name is prefixed with `basic_`, it's the template class; if it has a "w" in it, it's the multi-byte version; otherwise it's the ordinary class.
Besides the classes derived from `iosbase`, the library also includes classes for managing data buffers:

```
                       basic_streambuf
                      ________|________
                     |                 |
               basic_stringbuf     basic_filebuf
```

The virtual template class `basic_streambuf`, which is part of the library but not of the `ios_base` lineage, holds the data and functions needed to manage a character buffer.
Its derived classes `basic_stringbuf` and `basic_filebuf` are instead specialized in handling in-memory and on-file buffers, respectively.
Here too, the library includes two versions of each class, specialized for handling `char` and `wchar_t`.

```
typedef streambuf  basic_streambuf<char>
typedef wstreambuf basic_streambuf<wchar_t>
typedef filebuf    basic_filebuf<char>
typedef wfilebuf   basic_filebuf<wchar_t>
```

As you've probably guessed, going through the individual classes of the `iostream` library is an activity that rivals classical epigraphy for sheer tedium, but it'll let us see a whole series of principles we discussed in previous lessons put into practice, so, let's brace ourselves and get started.

The <code id="ios-base">ios_base</code> class and its first descendant, <code id="basic-ios">basic_ios</code>, are generic classes that provide the basic functions for managing streams, regardless of whether they're input or output streams.
A peculiarity of `ios_base` is that it has no public constructor, so it cannot be used to create objects — only as a base for derived classes.
The specialized instances of `basic_ios` are:

```
typedef basic_ios<char>    ios;
typedef basic_ios<wchar_t> wios;
```

Through the methods of these classes, you can check or change a stream's internal state, its formatting, or define callback functions for handling data.
The `openmode` member data, for example, defines how the stream should be opened:

|---|---|
|*app*    | Makes every output operation happen at the end of the stream.
|*ate*    | On opening the stream, moves the insertion point to the end (_**at e**nd_) of the I/O buffer.
|*binary* | Treats the stream's content as a binary data stream.
|*in*     | Allows input operations.
|*out*    | Allows output operations.
|*trunc*  | Clears the stream's content when opened.

The `iostate` member data, which we'll use in one of the next examples, holds information about the stream's current state:

|---|---|
|*goodbit*  | No error
|*eofbit*   | The end of the stream has been reached.
|*failbit*  | The last I/O operation failed.
|*badbit*   | The last I/O operation was invalid.
|*hardfail* | An unrecoverable error occurred.

Both of these member data items are bitmasks,<a class="nota" href="/en/notes#fn-bitmask" id="bitmask">1</a> so they can hold more than one value at once.
The following instruction, for example, opens a file stream by OR-combining three possible `openmode` values:

```
fstream file_io("io.txt"
               , ios_base::in | ios_base::out | ios_base::app);
```

After `basic_ios`, the library's classes specialize into input or output: on one side, `basic_istream`, from which the two standard input streams `cin` and `wcin` derive; on the other, `basic_ostream`, from which the standard output streams `cout`, `cerr`, `clog`, and their "wide" counterparts `wcout`, `wcerr`, `wclog`, derive.
From these two generic classes derive template classes specialized in input or output to file or memory:

```
template
<class Elem, class Tr = char_traits<Elem>>
class basic_ifstream
: public basic_istream<Elem, Tr>

template
<class Elem, class Tr = char_traits<Elem>, class Alloc = allocator<Elem>>
class basic_istringstream
: public basic_istream<Elem, Tr>

template
<class Elem, class Tr = char_traits<Elem>>
class basic_ofstream
: public basic_ostream<Elem, Tr>

template
<class Elem, class Tr = char_traits<Elem>, class Alloc = allocator<Elem>>
class basic_ostringstream
: public basic_ostream<Elem, Tr>
```

and a class able to handle both operations:

```
template
<class Elem, class Tr = char_traits<Elem>>
class basic_iostream
: public basic_istream<Elem, Tr>
, public basic_ostream<Elem, Tr>
```

this one too, with two specializations for handling files and memory:

```
template
<class Elem, class Tr = char_traits<Elem>, class Alloc = allocator<Elem>>
class basic_stringstream
: public basic_iostream<Elem, Tr>

template
<class Elem, class Tr = char_traits<Elem>>
class basic_fstream
: public basic_iostream<Elem, Tr>
```

Another feature of the `<<` and `>>` operators is that their precedence is lower than almost all other operators, which lets you write instructions like this one:

```
cout << "Due più due fa: " << 2 + 2 << '\n' ;
```

The logical AND operator `|`, the inclusive OR operator `&`, and the exclusive XOR operator `^` have lower precedence than the `<<` and `>>` operators, and, if not enclosed in parentheses, operations involving them can cause bugs.
For example, in an instruction like the following, the `&` operator would be interpreted as a reference to an object, with consequences different from those expected:

```
cout << "Il valore è: " << 2 & 2 << '\n' ;	// ERRORE!
```

the correct syntax, instead, is:

```
cout << "Il valore è: " << (2 & 2) << '\n' ;	// OK
```

The default behavior of stream input operators also includes some formatting conventions: <!-- @todo:verificare -->
- **the base conversion format is decimal**;
- **the fill character is a space**;
- **the precision of floating-point digits is the same used by `printf()`**, rounding at the sixth decimal digit;
- **the default field width is 0**, which means the output stream will use as many characters as needed to display the entire value or string.

The first three changes are permanent: once set, the new values remain in effect until another instruction changes them again; changes to the input field width, on the other hand, only apply to the instruction that requests them.
In some earlier examples we saw that it's possible to change a stream's default output format by means of *manipulators*:


<hr id="iostate">

When a stream read or write operation fails, the value of the `iostate` member data becomes non-zero.
The `basic_ios` class has boolean member functions that return `true` or `false` depending on whether the `iostate` value indicates a given event, and the `rdstate` function, which returns `iostate`'s absolute value.

|---|---|
| good    | no error: the value of `iostate` is 0
| eof     | the end of the file has been reached
| fail    | a non-blocking I/O error occurred
| bad     | a blocking I/O error occurred
| rdstate | returns the current value of `iostate`

These functions let you stop reading or writing a stream when an error occurs, or when the end of the file has been reached.

*Exceptions* let you handle errors that occur while the program is running.
When something abnormal happens, the system *throws* an exception, that is, it transfers control of the process from the current function to specific blocks of instructions called *exception handlers*.
For this to happen, the code that generates the error must be enclosed in a `try`/`catch` block:

```
try {

    // codice che potrebbe dare errore

} catch (...) {

    //  istruzioni per la gestione dell'errore
}
```

The C++ standard library has a dedicated class for handling exceptions:

```
class exception {
public:
    exception () throw();
    exception (const exception&) throw();
    exception& operator= (const exception&) throw();
    virtual ~exception() throw();
    virtual const char* what() const throw();
};
```
