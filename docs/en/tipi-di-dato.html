Data types

Everything is a byte

The variety of available data types is C++’s main distinguishing feature.

C++ inherited a wide range of data types from C. Depending on their nature, we can divide these data types into two groups: scalar data and aggregate data. The scalar data types are: void, bool, char, wchar_t, int, float, double, and they let you handle a program’s classic elements: integers, decimal numbers, letters, and boolean values. The aggregate data types are arrays, unions, enums, structs, and classes. These, as their name suggests, are made up of the aggregation of several scalar data items of the same type or of different types. The next piece of code — annoyingly didactic, in its essential uselessness — shows the main data types of C++. It contains many elements I haven’t told you about yet, so don’t worry if you don’t fully understand what it does.

#include <iostream>

using namespace std;

/** Definisce un tipo di dato enumerato di nome RGB */
enum RGB { red = 0xFF0000, green = 0x00ff00, blue = 0x0000ff };

/** Definisce una struttura che contiene un colore RGB e un nome */
struct ColoreRGB {
    RGB  valore;
    const char* nome;        
};

/** 
 * Definisce la classe Colore, che contiene un coloreRGB 
 * e una funzione che ne mostra il nome.
 */
class Colore {
public:
    ColoreRGB coloreRgb;        
    void nome_colore() {
        cout << "coloreRGB:";
        switch(coloreRgb.valore) {
            case red  : cout << "red"  ; break;
            case green: cout << "green"; break;
            case blue : cout << "blue" ; break;
        }
        cout << endl;
    }        
};

int main()
{    
    /** Dichiara una serie di variabili */
    bool    booleano  = false;
    char    carattere = 'C';
    int     intero    = 1234567890;
    float   decimale  = 3.14;
    char    array[]   = "abcdefghilmnopqrstuvz";
    RGB     enumerato = green;
    
    /** Crea un oggetto di tipo Colore */
    Colore  colore;    
    
    /** 
     * Assegna un valore ai dati della struttura coloreRgb 
     * all'interno dell'oggetto di tipo Colore.
     */
    colore.coloreRgb.valore = enumerato;   
    colore.coloreRgb.nome = "verde";     

    /** Mostra il valore delle variabili */
    cout << "booleano:"    << booleano   << endl;
    cout << "carattere:"   << carattere  << endl;
    cout << "intero:"      << intero     << endl;
    cout << "decimale:"    << decimale   << endl;
    cout << "array:"       << array      << endl;
    
    /** Mostra il nome del colore */
    colore.nome_colore();
    
    return 0;
}

The void data type has neither a size nor a value, and can only be assigned to a pointer or to a function that returns no value at all, like the nome_colore function of the Colore class. The bool data type is used to handle boolean values. It can only take two values, true or false, so it has the smallest possible size, namely one byte. The int type lets you handle integers. It’s usually four bytes in size, which can be increased or decreased by means of keywords called modifiers. The float and double types are used to handle decimal numbers. They are four and eight bytes long respectively, with a precision of seven and fifteen decimal digits. The three elements that appear outside the main function are aggregate data types, and let you define non-standard data types. enums let you assign names to values, making programming easier. In the example, the RGB enum assigns a name to three values of the RGB encoding. This new data type is used in the ColoreRGB struct. structs let you create sets of heterogeneous data. The ColoreRGB variable contains two different variables: a value of type RGB and a pointer to a character. All the data types we’ve seen so far are a legacy of the C language. The Colore class, on the other hand, is one of the innovations introduced by C++, and contains two elements: a struct of type ColoreRGB and a function that displays the color’s name. Be careful, though: none of these fellows is really what it claims to be. The booleano variable doesn’t actually contain the value true or false, but a binary value that the system treats as such. Likewise, the carattere variable doesn’t contain the letter C, but the binary value corresponding to the ASCII encoding of the letter C. Every data type is nothing more than the name of a quantity of memory. Remember this, because it’s the key to everything.

The amount of memory associated with each data type isn’t fixed, but can vary depending on the system you’re working on. The code below lets you find out the size, in bytes, of the scalar data types:

/** 
 * @file tipi-di-dato-dimensione.cpp
 * Mostra la dimensione dei principali tipi di dato del C++.
 */
 
#include <iostream>
#include <iomanip> 

using namespace std;

int main()
{    
    int w = 12;
    
    cout << setw(w) << "bool: "   << sizeof(bool)   << endl;
    cout << setw(w) << "char: "   << sizeof(char)   << endl;
    cout << setw(w) << "int: "    << sizeof(int)    << endl;
    cout << setw(w) << "float: "  << sizeof(float)  << endl;
    cout << setw(w) << "double: " << sizeof(double) << endl;
    
    return 0;
}

The number of possible values a data type can hold depends on its size. A char is made up of eight bits, and can therefore take 28 values, that is: from 0 to 255. An int is made up of thirty-two bits, and can take 232 values, that is: from 0 to 4,294,967,295. A float has 8 bits for the exponent and 23 for the mantissa, so it can take values from 1.175494e-38 to 3.402823e+38, and so on. Besides data types, C++ has also inherited from C the so-called modifiers, keywords which, when added to the name of a data type, change its characteristics. The short and long modifiers change the size of the data, shrinking or growing it. The signed and unsigned modifiers determine whether the first bit of the data is treated as its sign: positive if the value is 0, negative if the value is 1. The sign, however, “steals” a bit from the signed variable, which can therefore hold fewer values than the same unsigned variable. This program shows how the size and the minimum and maximum values of the int data type can change depending on which modifiers are used:

/** 
 * @file tipi-di-dato-limiti.cpp
 * Mostra i valori possibili per i principali tipi di dato del C++.
 */

#include <iostream>
#include <iomanip> 
#include <limits>

using namespace std;

/**
 * Funzione template che mostra la dimensione e i 
 * valori minimi e massimi per un tipo di dato.
 */
template<typename T>
void dimensione()
{
    cout << sizeof(T) << " bytes,"
         << setw(4)   << "da:"   
         << setw(21)  << numeric_limits<T>::min()
         << setw(4)   << "a:"    
         << setw(21)  << numeric_limits<T>::max() 
         << endl;
}

int main()
{    
    /** Variazione della dimensione e dei valori del tipo int */
    cout << setw(16) << "int: "; 
    dimensione<int>();
    cout << setw(16) << "unsigned int: "; 
    dimensione<unsigned int>();
    cout << setw(16) << "short int: "; 
    dimensione<short int>();
    cout << setw(16) << "unsigned short: "; 
    dimensione<unsigned short>();
    cout << setw(16) << "long int: "; 
    dimensione<long int>();
    cout << setw(16) << "unsigned long: "; 
    dimensione<unsigned long>();
    
    return 0;
}

The last thing I need to tell you, about C++’s data types, are the so-called intermediary attributes: functions, arrays, pointers, and references. We’ll talk about functions at length later on. What matters to us here and now is that every function has its own data type. The main function, which is the main function of every program, has data type int, and indeed it ends with the instruction:

return 0;

The nome_colore function, inside the Colore class, is of type void and, as you can see, has no return value. Arrays are sets of homogeneous data, and are declared by adding square brackets to the variable’s name. An array’s size must be defined at the moment it’s declared; this can happen either explicitly, by putting the number of elements inside the square brackets:

char elementi[10];

or by assigning the array a value that will determine its maximum size, as we saw in the first example:

char array[] = "abcdefghilmnopqrstuvz";

Each element of the array is identified by an index ranging from 0 to the number of elements in the array minus one. In the case of the array above, the letter a will have index 0, b will have index 2, and so on, up to z, which will have index 20.

To identify the values within the bit sequence, you need two pieces of information: the data type you’re pointing at, and its address in memory. Pointers let you obtain exactly this information: their value defines the memory address to read from; their type defines the size of the value being pointed at.

unsigned char * ptr = valori;

Many people think of pointers as little flags letting you mark a specific spot in memory, but this is only true of void pointers, which have no associated data type. Other pointers are less like a flag and more like a kind of mask that can slide along the bit sequence, isolating and highlighting the individual values that make it up:

As you can see, the bits are the same, but by changing the data type associated with the pointer, and therefore its size, the value of the data becomes something else, and the conversion of numbers into letters loses its meaning. The p1++ and p2++ instructions increment the pointer by one, that is, they move it forward in memory by a number of bytes equal to the size of the data type it’s associated with. The p1 pointer is an unsigned char, so it moves by eight bits; the p2 pointer is a short int, so it moves by sixteen bits. This is why the second reading loop takes half as long as the first: because the amount of bits read per iteration is doubled. If I’d been careless and copied the first reading loop’s code as-is, without dividing the number of iterations by two, the pointer would have kept “walking” forward in memory, past the boundaries of the array, and this, as we’ll see later, is definitely not good. References are aliases for the variables they’re associated with. As with gibbons or swans, the bond between a reference and its variable lasts a lifetime. While pointers can have a NULL value and can be associated with different variables over the course of processing, references must always be associated with a variable, and that association can never be changed afterward.

/** 
 * @file tipi-di-dato-reference.cpp
 * Utilizzo delle reference.
 */
 
#include <iostream>
#include <iomanip> 

using namespace std;

int main()
{    
    /** Definisce una variabile di tipo int */
    int  v = 10;     

    /** Definisce una reference alla variabile v */
    int& r = v;    

    /** I valori di v e r sono identici perché r è un'alias di v */
    cout << "valori iniziali: v=" << v << ", r=" << r << endl;

    /** Modifica il valore della variabile r */
    r = 20;

    /** Il valore di v sarà modificato di conseguenza */
    cout << "valori finali:   v=" << v << ", r=" << r << endl;
    
    return 0;
}

We’ll go deeper into the pros and cons of references when we talk about functions.

Since each data type is nothing more than a way of looking at memory, it’s possible to convert from one type to another. Conversions can be implicit or explicit. Implicit conversions are the ones that happen when a value is copied between variables of compatible type:

int  i = 0;
char c = i;

In this case, we assign to a variable of type char the value of a variable of type int, and the compiler lets us do it, because the value of the integer can be stored without any trouble in the single byte of the char. If we wanted to make this conversion explicit, we would have to use the syntax:

int  i = 0;
char c = (char) i;
char c = char (i);

The two forms are equivalent. The first is the one C++ inherited from C; the second is called functional notation. In the earlier example, the valori variable is a pointer to unsigned char, but we told the system to treat it as a pointer to short int with the instruction:

duebyte * p2 = (duebyte*)valori;

C++ has other ways of converting one data type into another too, but since they apply mainly to classes, we’ll see them later on.

Spazioni also define the limits of our quantities and units of measure. Think of a line: theoretically, it’s an infinite set of points extending in two directions. Points, however, have no dimension, so producing even a small segment of a line requires an infinite number of them. In essence, according to mathematicians, two forms of infinity coexist within every line: the syncategorematic infinity of length, and the categorematic infinity of the sequence of points. (Or maybe it’s the other way around: I can never remember which is the actual infinite and which the potential one.) You can plainly see that this is nonsense: the idea that multiplying nothing by infinity can get you something is just an attempt by mathematicians to justify their sex lives. The line is the ideal representation of a finite set of contiguous spazioni. It’s very, very thin, but it does have a dimension. Even the plane, with all due respect to René Guénon, is made up of a finite set of contiguous spazioni, and through a single point there don’t pass infinite lines, but a finite number, however high, because the granularity of the spazioni limits the number of possible angles. The same holds for the natural numbers: they can’t be extended to infinity, only up to the total number of active spazioni in the Universe, because any number beyond that would indicate a quantity that doesn’t exist, and would therefore be meaningless. Spazioni also define the smallest unit of time, what Master Canaro jokingly called “spazione-time”: the interval of time a spazione needs to acquire all the energy of one of the spazioni it’s in contact with. This is why the speed of light cannot exceed three hundred million meters per second: because it’s limited by the time it takes for energy to transfer between spazioni.
