C++

Ask not what the operating system can do for you; ask what you can do for the operating system.

In one of his books, Bjarne Stroustrup defined C++:

A programming language for doing non-trivial tasks.

Let the same be said of you.

C++ is a programming language created by Bjarne Stroustrup in 1983, while he was working at AT&T Bell Labs. Ten years earlier, his colleague Dennis Ritchie had created a programming language called C. C, as I’ve told you, was extremely effective if you had to program computers, but — like some of your fellow brothers — it didn’t handle real-life entities nearly as well. Stroustrup, who during his doctorate had worked with an object-oriented language called Simula67, thought that if he could add to the execution speed of C the ability to create new data types like Simula’s, he would get the perfect language. He was right. The name C++ refers to the ++ operator, used to increment the value of a variable by one:

/**
 * @file src/cplusplus-increment.cpp
 * Example use of the increment operator.
 */

#include <iostream>

using namespace std;

int main(int argc, char** argv)
{
    int C = 12;
    cout << "C value = " << C << endl;
    C++;
    cout << "C value = " << C << endl;
    return 0;
}

If you compile and run this code, you’ll get:

> g++ cplusplus-increment.cpp -o ../out/esempio
> ../out/esempio
C value = 12
C value = 13

C++, in fact, wasn’t a new language: it was an improved C. All the code and experience that had been built up around C until then could also be applied to C++. Paraphrasing Neruda, Stroustrup did to C what Jesus did to Judaism: just as the Nazarene took a stern religion, suited to a people fleeing through the desert, and improved it, making it less authoritarian, the Dane transformed a language conceived solely for managing computers into a language that could manage anything.

The main additions C++ brought to C are: data abstraction, object-oriented programming, and generic programming. Let me explain what these are now, but don’t worry if you don’t fully understand: we’ll go deeper into all these concepts later. C’s data types are:

char, int, float, double, array, enum, struct, union

More than enough to write the Unix kernel, but decidedly inadequate for writing a program to manage phone lines. For a programming language to handle a data stream, a user, a phone line, or even a horse-breeding farm with equal ease, it needs to be able to manage, in addition to its predefined data types, new entities defined by the programmer. This is what’s called data abstraction, and C++ achieves it by means of classes. Classes are the in-code representation of an entity:

class Horse {
private:
    Sex    _sex;
    string _breed;
public:
    const char* getSpecies() const {
        return "horse";
    }
    const char getSex() const {        
        return (char)_sex;
    }
    const char* getBreed() const {
        return _breed.c_str();
    }
    Horse() {}
    Horse(const char* breed, Sex sex ) {
        _breed = breed;
        _sex   = sex;
    }
};

or of a concept:

class Mating {
private:
    Horse _male;
    Horse _female;
    Date  _day;
public:
    Mating(Horse& male, Horse& female) {
        _male   = male;
        _female = female;
        time(&_day);
    }
};

Thanks to classes, the programmer can create new data types and use them within their program the same way they would use the language’s primitive data types. Each class has attributes and methods. Attributes are data that describe the characteristics of the class, for example, a horse’s breed or sex. Methods are functions that define how the class can interact with the other elements of the program. In the classes in the example, the attributes are the elements you see in the private section, while the methods are the functions you see in the public section. The function with the same name as the class is called the constructor of the class, because it “explains” to the compiler how objects of this class should be created. Classes, however, are the recipe, not the dish. To be used, classes must be instantiated into objects:

int main()
{
    Horse   stallion("Lipizzaner", male);    
    Horse   mare("Maremmano", female);
    Mating  mating(stallion, mare);
    cout << mating << endl;
    return 0;               
}

stallion, mare, and mating are three objects. The first two are instances of the Horse class, the third is an instance of the Mating class. If you add a bit of code to the classes we saw earlier and compile the program, you’ll get:

% g++ cplusplus-class.cpp -o ../out/esempio
% ../out/esempio                               
DATE: Sun Apr  5 10:38:31 2020
MALE: Species:horse, Sex:m, Breed:Lipizzaner
FEMALE: Species:horse, Sex:f, Breed:Maremmano```

For a programming language to call itself object-oriented, though, besides classes it must also be able to handle inheritance and polymorphism. Inheritance allows you to define new classes as an extension of existing classes: 

class Animal {
private:
    string _species;
    string _breed;
    Sex    _sex;
public:
    const char* getSpecies() const {
        return _species.c_str();
    }
    const char getSex() const {        
        return (char)_sex;
    }
    const char* getBreed() const {
        return _breed.c_str();
    }
    Animal() {}
    Animal(const char* species, const char* breed, const Sex sex ) {
        _species = species;
        _breed   = breed;
        _sex     = sex;
    }
};
class Horse : public Animal {
public:
    Horse() {}
    Horse(const char* breed, const Sex sex )
    : Animal("Horse", breed, sex ) {
    }
};

In the example above, we first defined an Animal class, which has three attributes: species, breed, and sex; then we defined a Horse class, deriving it from the Animal class. This way, if besides horses our program also had to handle other hoofed animals, we wouldn’t have to repeat the same instructions in each class, but could reuse those of the base class:

class Donkey : public Animal {
public:
    Donkey() {}
    Donkey(const char* breed, const Sex sex )
    : Animal("Donkey", breed, sex ) {
    }
};

At this point, your sharp mind should have caught a possible problem (assuming you’re paying attention, which I’m not entirely convinced of): the Mating class can only handle objects of type Horse. A language without polymorphism would force us to write two new classes: one for mules and one for hinnies:

class MatingMule {
private:
    Donkey _male;
    Horse  _female;
    Date   _day;
public:
    MatingMule(Donkey& male, Horse& female) {
        _male   = male;
        _female = female;
        time(&_day);
    }
};
class MatingHinny {
private:
    Horse  _male;
    Donkey _female;
    Date   _day;
public:
    Mating(const Horse& male, const Donkey& female) {
        _male   = male;
        _female = female;
        time(&_day);
    }
};

The problem is that the more code you write, the more likely you are to make mistakes, and the harder they’ll be to fix. By contrast, programs with fewer lines of code are more reliable and easier to fix or modify. C++ helps here because it allows for polymorphism, that is, the ability of a function or operator to carry out its task independently of the data type it has to handle. If we rewrite the Mating class using, instead of parameters of type Horse, parameters of the base class type Animal:

class Mating {
private:
    Animal* _male;
    Animal* _female;
    Date    _day;
public:
    Mating(Animal* male, Animal* female) {
        _male   = male;
        _female = female;
        time(&_day);
    }
};

We can create objects of class Mating with any derived class whatsoever:

/**
 * @file src/cplusplus-polymorphism.cpp
 * Example of class polymorphism.
 */

#include <iostream>
#include <ctime>

using namespace std;

typedef time_t Date;

typedef enum _sex {
    male = 'm',
    female = 'f'
} Sex;

class Animal {
private:
    string _breed;
    Sex    _sex;
public:
    virtual const char* getSpecies() const {
        return "";
    }
    const char getSex() const {
        return (char)_sex;
    }
    const char* getBreed() const {
        return _breed.c_str();
    }
    Animal() {}
    Animal(const char* breed, const Sex sex ) {
        _breed  = breed;
        _sex  = sex;
    }
};

ostream& operator << (ostream& os, const Animal& animal) {
    os  << "Species:" << animal.getSpecies() << "\t"
        << "Breed:"  << animal.getBreed()  << "\t"
        << "Sex:"  << animal.getSex()  << endl;
    return os;
}

class Horse : public Animal {
public:
    const char* getSpecies() const {
        return "Horse";
    }
    Horse() {}
    Horse(const char* breed, const Sex sex )
    : Animal(breed, sex ) {
    }
};

class Donkey : public Animal {
public:
    const char* getSpecies() const {
        return "Donkey";
    }
    Donkey() {}
    Donkey(const char* breed, const Sex sex )
    : Animal(breed, sex ) {
    }
};

class Mating {
private:
    Animal* _male;
    Animal* _female;
    Date    _day;
public:
    Mating(Animal* male, Animal* female) {
        _male = male;
        _female = female;
        time(&_day);
    }
    friend ostream& operator << (ostream& os, const Mating& mating) {
        os << "DATE: "    << asctime(localtime(&mating._day))
           << "MALE: " << *mating._male
           << "FEMALE: " << *mating._female;
           return os;
     };
};

int main()
{
    Animal* horse  = new Horse("Lipizzaner", male);
    Animal* mare   = new Horse("Maremmano", female);
    Animal* donkey = new Donkey("Amiata", male);
    Animal* jenny  = new Donkey("Sardinian", female);

    Mating foal(horse, mare);
    Mating jack(donkey, jenny);
    Mating mule(donkey, mare);
    Mating hinny(jenny, horse);

    cout << "FOAL\n"  << foal  << endl;
    cout << "JACK\n"    << jack    << endl;
    cout << "MULE\n"     << mule     << endl;
    cout << "HINNY\n" << hinny << endl;

    return 0;
}

As I’ve told you, though, a good programmer isn’t satisfied just because they’ve managed to produce the expected result — they always ask whether there’s a more efficient way to get there. In our case, the code we used to display the results of the matings:

Mating foal(horse, mare);
Mating jack(donkey, jenny);
Mating mule(donkey, mare);
Mating hinny(jenny, horse);

cout << "FOAL\n"  << foal  << endl;
cout << "JACK\n"  << jack  << endl;
cout << "MULE\n"  << mule  << endl;
cout << "HINNY\n" << hinny << endl;

isn’t the most efficient, both because we could get the parents’ species mismatched with the offspring’s name, and because the instructions have to be repeated for each object. To fix the first flaw, we can add to the Mating class an attribute and a method to independently determine what kind of offspring results from the mating: 

string   _outcome;
void setOutcome() {
    if(strcmp(_male->getSpecies(),"Donkey") == 0) {
        if(strcmp(_female->getSpecies(),"Donkey") == 0){
            _outcome = "donkey";
        } else {
            _outcome = "mule";
        }
    } else {        
        if(strcmp(_female->getSpecies(),"Horse") == 0) {
            _outcome = "foal";
        } else {
            _outcome = "hinny";
        }
    }
}

but even so, we’d still have to rewrite four lines of code to change the program’s output: an unacceptable approach for production systems, where the entities being handled can number in the thousands. We can solve this problem thanks to generic programming and the way it’s implemented in C++: template classes:

template < class T> class list;

The list class is one of C++’s template classes, and lets you insert, remove, move, merge, sort, and list a collection of objects of the same class. The syntax for creating a list of Mating-class objects is:

list<Mating> matings;

Once that’s done, we can add elements to our list with the push_back() function, passing it the class constructor directly:

matings.push_back(Mating(horse, mare));
matings.push_back(Mating(donkey, jenny));       
matings.push_back(Mating(donkey, mare));     
matings.push_back(Mating(horse, jenny));

This code is a more efficient form of:

Mating m1(horse, mare);
Mating m2(donkey, jenny);       
Mating m3(donkey, mare);     
Mating m4(horse, jenny);

matings.push_back(m1);  
matings.push_back(m2);  
matings.push_back(m3);
matings.push_back(m4);

To display the contents of the list, regardless of the number of elements, all it takes is the instruction:

for (list<Mating>::iterator it=matings.begin(); it!=matings.end(); it++) {
    cout << *it << endl;
}

Our program’s main() function will therefore be:

int main()
{
    Animal* horse  = new Horse("Lipizzaner", male);    
    Animal* mare   = new Horse("Maremmano", female);    
    Animal* donkey = new Donkey("Amiata", male);
    Animal* jenny  = new Donkey("Sardinian", female);
    list<Mating> matings;
    matings.push_back(Mating(horse, mare));
    matings.push_back(Mating (donkey, jenny));       
    matings.push_back(Mating (donkey, mare));     
    matings.push_back(Mating (horse, jenny));
    for (list<Mating>::iterator it=matings.begin();
         it!=matings.end(); it++) {
        cout << *it << endl;
    }
    return 0;               
}

Besides the predefined template classes of the Standard Template Library, C++ lets you define your own template classes, but we’ll get to that in due course. These features, combined with compatibility with existing C code, made C++ the most widely used object-oriented language of the 1990s.
